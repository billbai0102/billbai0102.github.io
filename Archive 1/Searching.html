SEARCHING

WHAT IS SEARCHING?
Searching is the act of locating a specific element within an Array, ArrayList, or another data structure. The two main searching algorithms used by programmers is the linear search and the binary search.
This lesson will be focusing on using the Linear Search and Binary search to locate an element within Arrays and ArrayLists.

LINEAR SEARCH
The linear search, sometimes also known as the sequential search, is the simplest and most straightforward search. It involves looping through each and every element in an array, and checking if the element is equal to the key.
It is typically performed on unsorted arrays or arrays with few elements. The following code below is that of the linear search.

Pseudocode

  1. Set x to 0
  2. If element at index x is equal to the key then return true, end loop or recursive function
  3. Increment x
  4. Go back to Step 2
Array Implementation ArrayList Implementation 
    //Linear Search for Arrays
    public boolean linearSearch(int arr[], int target){
        for(int x = 0; x < arr.length; x++){
            if(arr[x] == target)
                return true;
        }
        return false;
    }
    
    //Linear Search for ArrayLists
    public boolean linearSearch(List <Integer> arr, int target){
        for(int x = 0; x < arr.size(); x++){
            if(arr.get(x).equals(target))
                return true;
        }
        return false;
    }

The runtime for sequential search is O(n) time, meaning it is not very efficient.

Tips when using Linear Search

  • Due to it's horrible time efficiency, you should always implement a binary search instead, unless the array is unsorted.




BINARY SEARCH
Binary search is another search function. It is a lot more efficient than linear search, but one of it's drawbacks is that it can only be performed on sorted arrays.
In essence, it works by comparing the middle element of an array to the key. If the element is equal to the key, then it returns true. If not, it will compare the middle element of the subarray between the middle element and the leftmost or rightmost element of the old subarray, depending on whether its larger or smaller than the middle element, to the key, and continues this process until the key is either found, or there are no more elements to compare in the array.
The following code is an implementation of the binary search for Arrays and ArrayLists.

Pseudocode

  1. Create an int, low, to mark the leftmost element
  2. Create an int, high, to mark the rightmost element
  3. While high is bigger or equal to low,
  4. Create an int, middle, and set it to (low + high) / 2
  5. If element at index middle equals to the key, return true, end loop
  6. If the element at index middle is larger than the key, set high to middle - 1
  7. If the element at index middle is smaller than the key, set low to middle + 1
  8. Go back to step 3
Array Implementation ArrayList Implementation 
//Binary Search for Arrays
public boolean binarySearch(int[] arr, int key)
{
    int low = 0;
    int high = arr.length - 1;
    while(high >= low) {
        int middle = (low + high) / 2;
        if(arr[middle] == key) {
            return true;
        }
        if(arr[middle] < key) {
            low = middle + 1;
        }
        else if(arr[middle] > key) {
            high = middle - 1;
        }
    }
    return false;
}

The runtime for binary search is O(log n ) time. This means that for 1000 elements, it'd take a maximum of 10 iterations to find the element. Due to it's amazing efficiency, it is used for almost every search application.

Tips when using Binary Search

  • Although binary search may have an efficiency, make sure that you know the array that is being searched is sorted. If it is not sorted, it will be almost impossible to find the element in the array. If it is not sorted, use the linear search.
