OOP PART 2 - POLYMORPHISM

What's Polymorphism? Keep reading and you'll know!


In OOP Part 1, you learned about the first three principles of OOP - encapsulation, inheritance and abstraction. In this lesson, you will be learning about the last principle of OOP - Polymorphism.
Polymorhphism is the ability for an object to take on different forms. This refers to when a subclass takes on the reference of an object within it's parent hierarchy. This is called upcasting. As well, an upcasted object can also be downcasted back to one of it's original forms.

UPCASTING
As previously explained, upcasting and downcasting is when an object polymorphs from a subclass to a parent class, then to another parent class, or goes back down the inheritance tree back to itself. Take this inheritance tree,

With the concepts of polymorphism, that means the Vowel can upcast into a letter, character, or Object, while still retaining it's original data. Once it is upcasted, it can now be downcasted.
To check whether an object can be upcasted to another, use the "is-a" check taught in OOP - Part 1. As you can see, Vowel is-a letter, and Vowel is-a Character. This means you can successfully upcast. However, a Letter is NOT a Vowel, and a Vowel is NOT a digit, meaning you won't be able to successfully upcast between those two types. In Java, upcasting is done automatically. The basic outline of how to upcast is 

      ParentClass name = new Subclass();
      or
      ParentClass name = PreInstatiatedSubclass;

As well, objects are automatically upcasted to suit the parameters of a method.
Given this method, 

      public static void method(Letter letter){
        //code
      }

      public static void main(String[]args){
        Vowel vowel = new Vowel();
        method(vowel);
      }

Since vowel is-a letter, Java automatically upcasts vowel to an instance of Letter. This is a perfect example of upcasting polymorphism.

DOWNCASTING


Downcasting in Java is another big concept. It involves polymorphing an upcasted object back down to it's original form. 

1.    Vowel vowel = new Vowel(); //Creates Vowel object
2.    Letter upcastedVowel = vowel; //Upcast to Letter, because vowel is-a Letter
3.    if(upcastedVowel instanceof Vowel){
4.      Vowel downcastedLetter = (Vowel) upcastedVowel; //Reverts back to Vowel instance
5.    }
      //Now, downcastedLetter, upcastedVowel, and vowel are all instances of "Vowel".
      //Remember that although they have different names, they still point to the same
      //reference since new was only used once, and that was to create vowel.

There are a few things that went on in this example.
In line 1, a new Vowel object is created, named "vowel".
In line 2, the vowel is upcasted to Letter, but is now named upcastedVowel. Although the name has changed, it still refers to the same object, due to aliasing.
In line 3, there is a special if statement. The instanceof operator checks whether upcastedVowel is an instance of Vowel, which it is, since it was originally a Vowel instance.
In line 4, finally the upcastedVowel object is downcasted back to it's original Vowel instance, called downcastedLetter.

Another way to downcast temporarily to call a method of it's original form is by doing like this: 

      ((SubclassType)upcastedObject).performSubclassMethod();
so, for vowel, you'd do this:
      ((Vowel)upcastedVowel).performVowelMethod();

By doing so, you can call it's original type's methods while it is stil upcasted as a Letter. But once again, make sure that it is downcast-able, or a ClassCastException will be thrown.

Analysis
Upcasting and Downcasting may seem intimidating, but it is quite easy once you get the hang of it. Upcasting is done automatically, so no explicit cast is needed. If you enter a subclass into the parameters that ask for one of it's parent classes, then it will automatically be upcast. Although, if you do this vice versa, it will not work and you will need to explicitly downcast it. Upcasting and Downcasting is useful especially when you want to pass a subclass into the parameters that require one of it's parent classes. Image this: You have a Bird class with many subclasses. Birds all make the same sound, so you could create the method 

public void birdSound(Bird bird){
  System.out.println("Chirp");
}

By doing so, you can pass an Owl, a Turkey or any subclass of bird into it. They all make the same sound, so it would be more logical than creating instance methods for all the subclasses that all do the same thing. This is what upcasting and downcasting allow you to do.

PROBLEM: Knowing that a Bird is-an Animal, upcast the Bird to an animal, call the Animal "speak()" method, then the Bird "chirp()" method, then cast it back down

SOLUTION: 






Bird bird = new Bird(); //Create new bird
Animal upcastedBird = bird; //Upcast bird to Animal
bird.speak(); //Call the animal speak() method
((Bird) bird).chirp(); //Call the Bird chirp() method
Bird downcastedAnimal = (Bird) bird; //Downcast back to bird

How I solved this problem: First I constructed the Bird object, and named
it "bird". I then upcasted it to Animal, creating another alias to bird
called "upcastedBird". Since it is now an Animal, I used the speak() method
on bird. Remember, bird and upcastedBird all point to the same reference.
I then temporarily downcasted it to bird, to use the chirp() method for the
 Bird, then I actually downcasted bird, and created another alias called
 "downcastedBird" that also points to bird.
