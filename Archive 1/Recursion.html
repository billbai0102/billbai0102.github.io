RECURSION

WHAT IS RECURSION?
Recursion is the method of breaking a problem down into multiple smaller, easier to solve problems, then putting those small solutions together to answer the original problem. It involves a method calling itself.

INTRODUCTION
In programming, there are generally 3 types of recursion: Tail recursion, mutual recursion, and linear recursion.

Tail recursion is when a method recurs by calling itself at the end of the method. 

      int add5(int num, int times){
          num += 5;
          return add5(num, times + 1);
      }

As you can see, at the end of the method, the method calls itself again, without placing the method call in an if statement.

Mutual recursion is when a method recurs by calling another method which calls the original method. 

  void doSomethingA(int a){
    if(a <= 1)
      return a;
    doSomethingB(a - 1);
  }

  void doSomethingB(int b){
    doSomethingA(a - 2)''
  }

In this example, you can see that doSomethingA calls doSomethingB, which in turn calls doSomethingA again. This shows how mutual recursion uses another method to break down the problem into smaller pieces.

The last type of recursion is linear recursion. Linear recursion is a generic term for recursive methods that calls itself each time it runs. An example would be the Factorial Method. 

      int factorial(int n){
        if(n == 1){
          return 1;
        }
        return n * (fact(n-1));
      }

HOW DO I MAKE A RECURSIVE METHOD?
There are 3 simple criterias for a recursive solution. Firstly, it must have a base case, or a way to identify and deal with the endpoint of the solution. Secondly, there must be a way to continually reduce the calculation to smaller and more repetitive elements. Lastly, it must be able to obtain a larger component from the smaller elements created in the recursive process. 

          void recursiveMethod(parameters){
            if(base case satisfied){
              return something;
            }
            perform proccesses;
            recursiveMethod(parameters);
          }
This is how a recursive method should look.



REAL LIFE EXAMPLES OF RECURSION
An everyday example of recursion is fractals in nature. Fractals are infinite repeats of geometric shapes, but always decreases in size by a factor of X. It is represented by a pattern. In nature however, fractals are slightly different in the sense that it is not limited. This means that in nature, the basecase for the fractal recursive function is the size limit. The same pattern is repeated continually, until it reaches the size limit.

A beautiful example of recursion in nature.


Analysis

In summary, recursion is using a method to call itself, or recur, to break down a problem into many smaller problems, then reattaching the solutions to create a bigger solution. There is mutual, tail, and linear recursion, which all have different applications. But remember, when creating a recursive method, there are three key points: you need the base case, it must be able to break down a problem into smaller problems, and it needs to reattach smaller solution into a bigger solution!

TIPS
  • Although recursive solutions make problems simpler, do not overuse recursion. It can be very memory innefficient, since it is placing so many method calls in the stack.
  • Do not forget the base case! Otherwise, there will be StackOverflow Errors.
  • If you know the size stays constant, and you are working with primitive data types, it may be better to use an array, due to ArrayList inefficiencies when unwrapping wrapper classes
  • Do NOT use recursion for simple iterative problems, due to memory inneficiency.
  • Use recursion for algorithms that do not produce a linear tree. iterative solutions are best for linear trees.
  • Recursion is best used for Divive-And-Conquer algorithms


Problem: You must make a recursive solution that reverses the digits of an integer!

Solution: 

  

  

    

      
            /**
             * Reverses the digits of the int passed in the parameters.
             *
             * @param num The integer to be reversed
             * @return The reversed integer.
             */
            public static int revDigits(int num) {
                if (num < 10)
                    return num;
                return num % 10 * (int) Math.pow(10, (int) Math.log10(num)) + revDigits(num / 10);
            }

        How to solve this problem: This recursive method reverses the digits of an integer.
        It works by raising a number to the power of the log10 of the number on the opposite
        side, moving the number to the front. It then does this again with the
        following numbers, adding them together. This then recurs until it reaches the
        basecase, when num < 10, which just returns the num. This puts the first number
        in the last digit, successfully reversing the number inputted. It then returns
        this new reversed number.
