 Intro: Here we explain Java and Object-Oriented programming using the example of a superhero town. For a basic definition to help you understand the concept, think of Object-Oriented programming as a way of executing tasks with information provided by a reference to an object (remember, an object can be anything).

Every superhero town has a few key ingredients: headquarters, coffee bars, superheroes, and a couple of villains. A standard superhero town would have 5 headquarters and be located on planet Earth sometime around year 2500.

SuperheroTown
     has a certain number of headquarters
     a certain number of coffee bars
     has a certain number of superheroes
     has a certain number of villains
     is located somewhere
     exists at a certain time
a typical SuperheroTown would have
     number of headquarters = 5
     location = Earth
     time period = 2500 

public class SuperheroTown
{
     int headquarters;
     int coffeeBars;
     int superheroes;
     int villains;
     String location;
     int time;
     public SuperheroTown()
     {
          headquarters = 5;
           location = "Earth";
          time = 2500;
     }
}

The above class is a template for a SuperheroTown object, or a template for creating a Superhero town on Earth. This code tells us that any object of SuperheroTown, or that any superhero town on Earth, has a few key traits which can have different values. For example, different towns will have different number of coffee shops. In the code, int headquarters is a variable. The keyword int tells us that the variable will store an integer, and headquarters is the name of the variable.

In the code under public SuperheroTown(), we set the number of headquarters to 5 and the location to Earth for ALL superhero towns created by this template, because all superhero towns are on Earth and they all must have 5 headquarters. They also all exist in the year 2500. This code is written in the constructor, indicated by the keyword public, followed by the name of the class. The constructor is what creates an object based on the template and initializes values for the declared variables.

Now we know the key traits of a superhero town and the values of some of the traits when an object is created based on the class SuperheroTown. We can now go ahead and set other values for objects of class SuperheroTown. Then we will be able to manipulate these objects for our code to actually do something, to complete a task.

Let us begin the storyline, or the task that is to be done by the code.

The conflict at Starbucks coffee bar. This story takes place in a superhero town called Towntington. Towntington has five superheroes, thirty coffee bars, and five villains.

Main Conflict
     Superhero Town Towntington is a new Superhero Town.
     the number of coffee bars in Towntington is fifteen.
     the number of superheroes is five.
     the number of villains is five.

public class Conflict {
     public static void main (String arguments[]) {
          SuperheroTown towntington = new SuperheroTown;
          towntington.coffeeBars = 30;
          towntington.superheroes = 5;
          towntington.villains = 5;
     }
}

Our plot begins by creating setting some basic values to establish the setting of our storyline. An object of class SuperheroTown is created by using the keyword new as shown in the code. This would call the constructor, which creates the object. Some unique values are assigned to variables of towntington, which is the created object of SuperheroTown. All of this is done in class Conflict. Just as we needed a template for superhero towns on Earth, we need a template for conflicts in these towns. This class will contain some information which we refer to in developing our storyline. Using multiple classes helps keep code easy to read and organized.

Now let’s add another element to our story. Of course, we couldn’t have a superhero town without a superhero.

All humans start out with 2 legs, 2 arms and 1 nose. They are either male or female and they have a name. They also have a food that they carry on themselves. If asked for their name, they will reveal their name.

Human
     have a name
     have a certain gender
     have a food in their pocket
     have a certain number of legs
     have a certain number of arms
     have a certain number of noses
A standard human would start with
     2 legs
     2 arms
     1 nose
When asked for name
     say name

public class Human
{
     String name;
     String gender;
     String currentFoodOnThem;
     int legs;
     int arms;
     int noses;
     public Human() {
          int legs = 2;
          int arms = 2;
          int noses = 1;
     }
     public String whatIsYourName() {
          return name;
     }
}

Now we have a template to create humans. Every human will have some key traits. Some of these traits will have a definite, set value, such as number of legs. Other traits will have values assigned later because not all humans are identical. You can interact with the human by asking for a name. The name is returned in something called a return method. A return type is indicated when creating a return method. In this case it is a String. A return method doesn’t actually output anything, but gives back a value for the computer to use elsewhere. Methods are also known as behaviours.

We first create a class Human since our human superhero will inherit the traits and behaviours of a human. Now let’s create a superhero.

All superheroes start out with a cape, two gloves, a mask, and a certain muscle mass. They have different powers.

Superhero extends the idea of Human.
     have a certain number of capes
     have a certain number of gloves
     have a certain number of masks
     have a certain muscle mass
     have a unique super power
A standard superhero would start with
     one cape
     two gloves
     one mask
     20 lbs of muscle

public class Superhero extends Human{
     int capes;
     int gloves;
     int masks;
     int muscleMass;
     String uniqueSuperPower;
     public Superhero() {
          capes = 1;
          masks = 1;
          gloves = 2;
          muscleMass = 20;
     }
}

In the above code, we used extends to indicate that superheroes are a subclass of Human, and will inherit traits of Human in addition to having some other features. Once again we set some initial values and let the rest be set once an object is created from the template.

Here is another example of using extends. We are going to add a villain to our story.

Villains are based on the idea of humans. They are almost identical, the only thing setting them apart is that they have a mustache, a hat, a certain "look", some level of craziness, and a certain quantity of “prisoners” taken by them. The standard villain will look evil, start the day out not very greedy, and not yet have prisoners in hand.

Villain extends the idea of Human.
A Villain
     has a mustache.
     has a hat.
     has a "look".
     has some level of craziness.
     has a certain number of prisoners in their possession.
For a given Villain,
     He will look mean.
     He will start out with very little craziness.
     He will start the day without any prisoners.

public class Villain extends Human {
     String mustacheColor;
     String hatColor;
     String look;
     int craziness;
     int numberOfPrisoners;
     Human prisoner;
     public Villain() {
          look = "Mean";
          craziness = 1;
          numberOfPrisoners = 0;
     }
}

Nothing new here, except you may have noticed that we declared a variable as Human prisoner. Same concept, Human is the type and prisoner is the name of the variable. When you create a class, you also create a data type. A String is really an object of a String class and its own data type. You will later see that the variable prisoner is really a name used to identify another object of the Human class.

Let’s give our villain some behaviours. Creating methods (behaviours) is how we can manipulate and interact with objects to do a task. Take a close look at the next two examples of methods.

Whenever the main plot says that a villain puts his left over food into the wrong bin, his level of craziness will go up by one.

putFoodInRecycling
     craziness increases by one

putFoodInRecycling() {
     craziness++;
}

In this method, we add one to the value of craziness whenever our villain puts food in the recycling (villains are horrible, aren’t they?). We increment craziness by one by using ++ as shown above. This is a void method, meaning that it doesn’t return a value. The following is another method for our villain.

If someone asks a villain how crazy he is, the villain will always respond with his level of craze.

howCrazyAmI
     tell them how crazy I am

public int howCrazyAmI() {
     return craziness;
}

This is a return method. You’ve seen this before, with returning the name of a human. This is no different, except our return type is an int.

Now we have one more method for our villain. It isn’t a different type of method, but you will see how our villain interacts with another human. Translating that into Java terms, you’ll see how this next method will take in information about a created object.

If the villain is supposed to take in a prisoner, tie up the specified prisoner, add 1 to the number of prisoners he has taken, and then print out "Golly! [The specified prisoner] has been tied up!”

takePrisoner(name)
add one to the number of prisoners this villain has taken.
print "Golly! [The specified prisoner] has been tied up!”

public void takePrisoner(Human prisoner) {
this.prisoner = prisoner;
numberOfPrisoners++;
System.out.println("The Villain has tied up " + prisoner.whatIsYourName());
}

This method might seem a little confusing. In the brackets beside the method name, you see Human prisoner. This is a parameter. It will be passed when the method is called. What Human does when placed in those brackets is that it tells Java that a value is being passed and that the value is of type Human. The identifier prisoner tells Java what the variable name for that value should be called. Now the method takePrisoner has a local variable called prisoner of type Human. Local variables cannot be used outside of the method they are declared in.

They reserved word this is used to refer to the variable declared at the top of the class, outside any methods (called an instance variable). It is useful when there are local variables and instance variables with the same name, although you should generally avoid reusing identifiers (name of a variable, object, method, etc.) in code.

Whenever the villain takes a prisoner, the variable numberOfPrisoners increases. Now, there is output in this method (indicated by System.out.println), but this is different from returning a value, so this is a void method. In the output, there is a String, plus another String returned from a method in the Human class. Don’t get confused, prisoner is an object of class Human. By writing prisoner.whatIsYourName() we are doing two things. Firstly, we are saying that we want to reference code in the class that prisoner belongs to (this is class Human). Secondly, we are calling the method whatIsYourName(), which is located in class Human and returns a value when called. Since objects of the same class can still be unique, you must use prisoner. instead of Human. when making a reference . If you wanted all objects of a class to have the same values, you can use the keyword static. If you changed the value of any variable of one object, the value of that variable would change for all objects of that same class. We will not be explaining exactly how to use static in this tutorial.

In the code above, we had a main method. There should be only one main method out of all the classes used in this program. A main method simply tells Java what class to start from and is where the programmer makes use of behaviours in the classes to do a task. The behaviours, or methods, are called by making reference to an object belonging to the class where the behaviour is located. Now you can see how our storyline come all together. To understand OOP, simply remember template (class), object, and reference. Objects are made by referencing a template, which is like a recipe. Then the objects are referenced to access values and behaviours. The main advantage of having objects in programming is to be able to create as many related objects as wanted while allowing each to be unique in the values of their traits. Each object will have a lot of information behind the scenes which helps reduce code. So hopefully our superhero town has brought you a few steps closer to understanding OOP.
