SEARCHING

WHAT IS SEARCHING?
Searching is the act of locating a specific element within an Array, ArrayList, or another data structure. There are two main search functions: Sequential searching, and Binary searching.
This lesson will be focusing on using the Linear Search and Binary search to locate an element within Arrays and ArrayLists.


SEQUENTIAL SEARCH
The linear search, sometimes also known as the sequential search, is the simplest and most straightforward search. It involves looping through each and every element in an array, and checking if the element is equal to the key.
It is typically performed on unsorted arrays or arrays with few elements. The following code

Pseudocode

  1. Set x to 0
  2. If element at index x is equal to the key then return true
  3. Increment x
  4. Go back to Step 2
Array Implementation ArrayList Implementation 
    //Linear Search for Arrays
    public boolean linearSearch(int arr[], int target){
        for(int x = 0; x < arr.length; x++){
            if(arr[x] == target)
                return true;
        }
        return false;
    }
    
    //Linear Search for ArrayLists
    public boolean linearSearch(List <Integer> arr, int target){
        for(int x = 0; x < arr.size(); x++){
            if(arr.get(x).equals(target))
                return true;
        }
        return false;
    }

Tips when using Linear Search

  • When choosing a search algorithm, always choose to use Binary search over linear search, unless the list is unsorted. This is because binary search has a much better time efficiency.


The runtime for sequential search is O(n) time, meaning it is not very inefficient.


BINARY SEARCH
Binary search is one of the most efficient searches, but one of it's drawbacks is that it can only be performed on sorted Arrays or ArrayLists.
