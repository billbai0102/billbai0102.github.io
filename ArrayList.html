ARRAYLISTS

WHAT IS AN ARRAYLIST?
An ArrayList is very similar to vanilla arrays in the sense that they are also data structures that hold elements of a specific type.

ARRAYS VS ARRAYLISTS
The difference between ArrayLists and arrays are that ArrayLists have variable lengths, unlike the constant length of arrays. When the size of an ArrayLists isn't large enough to hold another element, it doubles it's capacity.
As well, ArrayLists can not hold primitive data types, but rather hold the primitive types held in wrapper classes. This is slightly less efficient that arrays in some situations, since it will take extra time to unwrap the wrapped primitive type, when proccessing.
Lastly, ArrayLists use methods to alter itself, rather than arrays where you access the variable itself to modify it.

An example of ArrayList size vs length.



HOW DO I MAKE AN ARRAYLIST?
ArrayLists are declared as such: 

     ArrayList<type> name = new ArrayList<type>()

That means, to declare a String array, you'd do as such: 

     ArrayList<String> stringList= new ArrayList<String>();

Alternatively, if you were to create an ArrayList of BankAccount objects, you declare it as such: 

     ArrayList<BankAccount> bankAccountList = new ArrayList<BankAccount>();



WHAT CAN I DO WITH THIS ARRAYLIST?
Just like arrays, you can alter an ArrayList. There are 4 methods to alter an ArrayList's state. 

Firstly, to add an element into an array, you use the add(E element) method.
      stringList.add("Hello");
This would add the String to the end of the ArrayList.
Alternatively, you can also use the overloaded add(int index, E element) method to add an object at a specific location.
      stringList.add("Hello", 1);
This would shift every element after index 1 to the right, and place "Hello" at index 1. HOWEVER - This will throw an error if the index exceeds the size of the ArrayList.


  
  The ArrayList add() method




As well, there is a set(int index, E element) method which replaces the element at index with the new element.
      stringList.set(1, "Bye");
This will replace the second element in the ArrayList with "Bye", and discard the old element.


  
  The ArrayList set() method



The method remove(int index) will remove the object at the specified index.
      stringList.remove(1);
This will remove the second element, and shrink the size by 1.
As well, there is another method remove(E element) which will remove the first occurence of the specified element, and shrink the size by 1.


  
  The ArrayList remove() method



Lastly, there is the size() method, which will return the number of elements in the array.
      System.out.println(stringList.size()); //Will return the number of elements in stringList.


  
  The ArrayList size() method. NOTE: The 0.0 is the default value for double types, meaning the 0.0 mark an empty position in the ArrayList.

WHEN DO I USE AN ARRAYLIST? Just like arrays, ArrayLists hold a collection data. One thing ArrayLists offer that arrays don't is dynamic size. Rather than creating a new array each time more data is added, by using an ArrayList, it auto adjusts for you.
So in essence, use ArrayLists when working with data with ever changing size
Common reasons to use ArrayLists are

  • Storing values to be sorted
  • In real life applications, it can hold a bank account information or
  • Store student information on a record
  • Arrays also allow you to search for a specific element, and perform an operation on it

ANALYSIS
ArrayLists are just like arrays, except a few qualities: firstly, ArrayLists use methods to get, set, and remove elements, while arrays only use the "=" operator to modify everything. ArrayLists are best used with dynamic-sized datasets that are always changing length. This is the one biggest advantage they have over arrays. Arrays are better for data sets that don't change size, and deal with primitive variables. This is because ArrayLists are less efficient than arrays when working with primitive variables since they need to wrap and unwrap primitive variables with their wrapper classes.



Tips when working with ArrayLists

  • Just like arrays, when finding the last element, remember that it is list.size() - 1
  • Don't forget ArrayLists are also PASS BY REFERENCE!
  • If you know the size stays constant, and you are working with primitive data types, it may be better to use an array, due to ArrayList inefficiencies when unwrapping wrapper classes
